# Terminal Map and Function Codes


---

## 1. What this application is

A multi-desk trading terminal focused on:

- **Volatility structure** (surface, smile, term)  
- **Greeks and option risk**  
- **Dealer-style positioning (GEX/DEX)**  
- **Rates / macro plumbing**  
- **Trade lab / simulation tools**  
- **Academy** (this curriculum)

It is an integrated shell: function codes + sections, not a single-vendor heatmap clone.

---

## 2. Major desks (conceptual)

| Desk | Role in the three-machine model |
|------|----------------------------------|
| **Vol / Home** | Price of risk — surface, smile, term, hist IV |
| **Positioning** | Chain, GEX, levels, tools |
| **Greeks / Trade analyze** | Risk stack, profiles, lab |
| **Rates** | Plumbing, curves, STIR, FX, auctions |
| **Crypto** | BTC/ETH options context when enabled |
| **Academy** | Education without leaving the shell |

## 4. Navigation habits

1. Learn **codes** for muscle memory (Bloomberg-like GO culture).  
2. Use **section rails** within a desk for deep work.  
3. Keep **symbol** context in mind—GEX for AAPL is not SPX.  
4. Trust **freshness badges**; delayed chains change education quality.  
5. Open **Explain / glossary** on any metric you cannot define in one sentence.

---

## 5. Suggested first-day tour (90 minutes)

1. `ACAD` — read *Three Linked Machines*.  
2. `RATES` → `PLUMB` — write today’s corridor regime.  
3. `SURF` + `SMILE` + `TERM` on a liquid ETF/index proxy.  
4. `GEX` — mark walls and net sign.  
5. `GRK` — ATM θ vs γ comparison.  
6. Return to Academy *Theta Is Not Free Money*.

---

## 6. Key takeaways

1. One shell, many desks, one mental model.  
2. Codes beat hunting menus.  
3. Academy is part of the product, not an external wiki only.  
4. Freshness and conventions matter as much as chart beauty.  
5. Tour with a journal, not only clicks.
